# SK-002: 实体溯源技能

## 基本信息

| 属性 | 值 |
|------|-----|
| 技能ID | SK-002 |
| 技能名称 | 实体溯源 |
| 英文名 | Entity Trace |
| 版本 | 1.0 |

## 触发条件

满足以下任一条件时触发：
- 查询中包含具体公司/产品/人物名称
- 需要了解某个实体的基本信息
- 需要追踪实体间关联关系
- 涉及投资/商业决策需核实实体背景

## 触发判断

```
用户查询 → 检查：
  ├── 包含公司名称（华为/苹果/英伟达）→ 触发
  ├── 包含人名（马斯克/奥特曼）→ 触发
  ├── 包含产品名（ChatGPT/GitHub Copilot）→ 触发
  ├── 包含地名/机构名 → 触发
  └── 其他 → 不触发
```

## 执行规则

### Step 1: 实体识别

识别实体类型：

| 实体类型 | 示例 | 溯源重点 |
|---------|------|---------|
| 公司 | 英伟达、字节跳动 | 业务、股价、近期动态、关联公司 |
| 人物 | 马斯克、黄仁勋 | 背景、职位、近期言论、关联公司 |
| 产品 | ChatGPT、MacBook | 功能、版本、竞品、所属公司 |
| 技术 | Transformer、HBM | 原理、版本、应用场景、发展历程 |
| 机构 | 斯坦福、OpenAI | 属性、产出、近期动态 |

### Step 2: 基础信息获取

必须获取的信息：
- 实体全称与简称
- 所属领域/行业
- 成立时间/背景
- 核心业务/产品
- 近期（3个月内）重大动态

### Step 3: 关联实体挖掘

查找该实体的：
- 母公司/子公司
- 合作伙伴/竞争对手
- 关键人物/创始人
- 投资关系/并购历史

### Step 4: 交叉验证

- 至少2个来源交叉核实关键信息
- 标注信息来源（官方/媒体/第三方）
- 标注数据时效性

## 输出格式

```markdown
## 实体溯源结果: [实体名称]

### 基本信息
| 项目 | 内容 |
|------|------|
| 类型 | [公司/人物/产品/技术] |
| 所属领域 | [领域] |
| 成立/发布 | [时间] |
| 核心信息 | [一句话描述] |

### 近期动态（3个月内）
- [时间1]: [事件1]
- [时间2]: [事件2]

### 关联实体
- [关联1]: [关系]
- [关联2]: [关系]

### 信息可信度
[高/中/低] — [来源说明]
```

## 典型场景

| 场景 | 原始查询 | 溯源重点 |
|------|---------|---------|
| 投资研究 | "英伟达最新消息" | 业务+财报+AI芯片动态+竞争对手 |
| 人物了解 | "黄仁勋是谁" | 背景+职位+英伟达关系+近期动态 |
| 产品选型 | "ChatGPT和Claude对比" | 产品功能+版本+母公司+最新动态 |
| 行业研究 | "OpenAI最近在做什么" | 组织架构+研究方向+融资+竞争格局 |

## 失效条件

连续2次溯源结果与事实不符：
1. 检查信息来源是否权威
2. 考虑是否存在信息滞后
3. 记录失败案例到 failure_log.md
