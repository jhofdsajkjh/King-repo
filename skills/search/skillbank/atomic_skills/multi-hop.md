# SK-006: 多跳递进检索技能

## 基本信息

| 属性 | 值 |
|------|-----|
| 技能ID | SK-006 |
| 技能名称 | 多跳递进检索 |
| 英文名 | Multi-Hop Retrieval |
| 版本 | 1.0 |

## 触发条件

满足以下任一条件时触发：
- 问题包含多个实体/事件，需要串联推理
- 答案不能从单一检索结果直接获得
- 包含"为什么"、"因为"、"所以"、"导致"等因果关系词
- 包含"先后"、"然后"、"接着"等顺序关系
- 需要先查A再查B才能回答

## 触发判断

```
用户查询 → 检查：
  ├── 包含因果连词（因为→所以）→ 触发
  ├── 包含递进词（先→再→最后）→ 触发
  ├── 多个实体需要关联分析 → 触发
  ├── 无法用单一检索式回答 → 触发
  └── 简单事实查询 → 不触发
```

## 执行规则

### Step 1: 问题拆解

将复杂问题拆解为子问题链：

```
原始问题 → [拆解分析]
  ├── Q1: [第一个子问题] → 检索目标
  ├── Q2: [第二个子问题] → 检索目标  
  ├── Q3: [第三个子问题] → 检索目标
  └── 答案整合: [整合逻辑]
```

### Step 2: 逐跳检索

按顺序执行检索：

| 跳次 | 检索内容 | 目的 |
|------|---------|------|
| Hop 1 | [子问题1] | 获取基础信息 |
| Hop 2 | [子问题2] | 在Hop1基础上深入 |
| Hop 3 | [子问题3] | 综合判断 |
| ... | ... | ... |

### Step 3: 信息拼接

将各跳结果按逻辑关系拼接：
- 时序关系：按时间顺序排列
- 因果关系：前因后果
- 包含关系：从宏观到微观
- 对比关系：并列分析

### Step 4: 验证完整性

检查：
- 每跳信息是否获取完整
- 各跳之间逻辑是否连贯
- 最终答案是否覆盖原问题所有维度

## 输出格式

```markdown
## 多跳检索结果

### 问题拆解
**原始问题**: [用户问题]

**拆解过程**:
- Hop 1: [子问题1] → [检索式]
- Hop 2: [子问题2] → [检索式]
- Hop 3: [子问题3] → [检索式]

### 各跳结果

**Hop 1: [子问题1]**
[检索结果摘要]

**Hop 2: [子问题2]**  
[在Hop1基础上的结果]

**Hop 3: [子问题3]**
[综合结果]

### 答案整合
[最终答案，按逻辑串联各跳信息]

### 置信度评估
[各跳置信度] → 综合置信度: [高/中/低]
```

## 典型场景

| 场景 | 原始问题 | 拆解逻辑 |
|------|---------|---------|
| 因果分析 | "为什么英伟达能引领AI芯片" | 1.英伟达技术优势 → 2.生态建设 → 3.市场地位 |
| 人物关系 | "马斯克为什么要创办xAI" | 1.马斯克背景 → 2.OpenAI关系 → 3.xAI目标 |
| 事件分析 | "为什么A股半导体板块近期大涨" | 1.政策面 → 2.资金面 → 3.业绩面 |
| 投资研究 | "中国软件是否值得持有" | 1.公司基本面 → 2.行业前景 → 3.竞争格局 → 4.估值 |

## 失效条件

- 某跳检索失败/无结果 → 标记该跳为"未确认"，尝试替代检索路径
- 连续2跳失败 → 回退到简单检索，输出部分结论
- 各跳逻辑矛盾 → 标注冲突，让用户判断
