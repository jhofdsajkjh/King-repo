# SK-001: 关键词扩写技能

## 基本信息

| 属性 | 值 |
|------|-----|
| 技能ID | SK-001 |
| 技能名称 | 关键词扩写 |
| 英文名 | Keyword Expansion |
| 版本 | 1.0 |

## 触发条件

满足以下任一条件时触发：
- 初始查询过于模糊、笼统
- 查询词少于3个实词
- 搜索结果返回大量无关内容
- 用户未提供限定词（时间/范围/角度）

## 触发判断

```
用户查询 → 检查：
  ├── 实词数量 < 3 → 触发
  ├── 包含"哪些"、"有什么"、"关于" → 触发
  ├── 搜索结果过去3次精度<3 → 触发
  └── 其他 → 不触发，直接检索
```

## 执行规则

### Step 1: 提取核心概念

从原始查询中识别：
- 主要实体（名词）
- 行为/状态（动词）
- 属性/限定（形容词/副词）

### Step 2: 概念扩展

使用以下维度扩展：

| 维度 | 示例 |
|------|------|
| 同义词 | 大模型 → LLM, 大语言模型, 生成式AI |
| 下位词 | 苹果 → iPhone, 苹果公司, Apple |
| 相关领域 | Python → 机器学习, 数据科学, 自动化 |
| 场景词 | 赚钱 → 副业, 投资, 被动收入 |
| 技术术语 | 搜索 → 检索, RAG, 语义搜索 |

### Step 3: 组合生成

生成3-5个关键词组合：

```
原始: "AI最新进展"
扩写:
  - "人工智能 大模型 2024 2025 最新"
  - "LLM GPT Claude 最新进展"
  - "生成式AI 突破 2025"
  - "AI 模型 评测 最新"
```

## 输出格式

```markdown
## 关键词扩写结果

### 原始查询
[原始用户查询]

### 核心概念
- [核心词1]: [扩展词1], [扩展词2]
- [核心词2]: [扩展词1], [扩展词2]

### 推荐检索式（按优先级）
1. [检索式1]
2. [检索式2]
3. [检索式3]

### 扩写置信度
[高/中/低] — [原因简述]
```

## 典型场景

| 场景 | 原始查询 | 扩写后检索式 |
|------|---------|-------------|
| 模糊技术查询 | "AI编程" | "AI编程 GitHub Copilot Cody 2024 2025" |
| 产品调研 | "有什么好用的AI工具" | "AI工具 2025 最新 免费 好用" |
| 趋势了解 | "今年芯片行业" | "芯片行业 2025 进展 AI芯片 GPU" |
| 人物了解 | "马斯克" | "马斯克 Tesla SpaceX Neuralink 2025" |

## 失效条件

连续2次扩写后检索精度 < 3时：
1. 检查扩写方向是否偏离
2. 考虑切换为 entity-trace 或 expert-query
3. 记录失败案例

## 关联技能

- SK-002 entity-trace（扩写后包含实体时优先）
- SK-007 expert-query（专业领域术语扩写）
- SK-003 time-filter（有时间线索时优先）
