# APEX 全域LLM统一适配公式体系

基于双阶推理架构+多维度量化评分，打通单机本地模型、API模型、串联组合模型，一套公式、一套范式兼容全栈LLM。

## 一、基础符号定义（全局统一）

| 符号 | 含义 | 取值范围 |
|------|------|----------|
| S<sub>total</sub> | 综合效能得分（优化目标，最大值为最优） | [0, 1] |
| A | 任务准确率/逻辑正确性 | [0, 1] |
| F | 输出格式、语法、语句流畅度 | [0, 1] |
| R | 内容与指令相关性、上下文匹配度 | [0, 1] |
| E | 推理效率（时延/吞吐） | [0, 1] |
| C | 资源开销（显存/CPU/调用成本） | [0, +∞) |
| α,β,γ,ε,δ | 维度权重系数 | α+β+γ+ε+δ=1 |
| λ | 本地模型硬件适配系数 | λ=0.95 本地, λ=1 API |
| η | 级联模型修正系数 | η=1 单模型, η=0.85 级联 |

## 二、约束规则

1. 所有权重和恒为 1，保证得分区间收敛在 [0,1]；
2. **C 为负向指标**（开销越大，得分越低）；
3. **λ 仅作用于本地部署LLM**，云端API模型 λ=1；
4. **η 针对级联模型**（推理模型+输出模型串联），补偿多轮调用损耗。

## 三、核心统一主公式

### 1. 通用基础公式（单模型/本地独立LLM）
```
S_total = λ · (αA + βF + γR + εE - δC)
```

### 2. 级联双模型公式
```
S_total = η · λ · (αA + βF + γR + εE - δC)
```

**核心特性**：一套公式覆盖所有形态，仅通过调整系数 λ、η 区分部署模式。

## 四、分场景系数配置

### 场景1：纯本地单LLM（Ollama/Llama.cpp/TGWebUI）

| 子场景 | α | β | γ | ε | δ | λ | η |
|--------|---|---|---|---|---|---|---|
| 代码/推理/运维 | 0.50 | 0.20 | 0.15 | 0.10 | 0.05 | 0.95 | 1 |
| 文案/对话/创作 | 0.30 | 0.40 | 0.20 | 0.08 | 0.02 | 0.95 | 1 |
| 低配置主机 | 0.35 | 0.20 | 0.15 | 0.25 | 0.05 | 0.90 | 1 |

### 场景2：本地双模型级联（本地DeepSeek-R1 + 本地轻量输出模型）

| 子场景 | α | β | γ | ε | δ | λ | η |
|--------|---|---|---|---|---|---|---|
| 代码级联 | 0.50 | 0.20 | 0.15 | 0.10 | 0.05 | 0.95 | 0.85 |
| 内容创作 | 0.30 | 0.40 | 0.20 | 0.08 | 0.02 | 0.95 | 0.85 |

### 场景3：本地LLM + OpenAI混合架构

| 配置 | 值 |
|------|----|
| λ | 1 |
| η | 0.90 |
| 权重 | 复用场景1/2对应业务权重 |

## 五、指标量化标准

| 指标 | 量化规则 |
|------|----------|
| **A (准确率)** | 代码：编译通过率/用例成功率；问答：答案正确率 |
| **F (流畅度)** | 文本：语法错误/格式完整性；代码：缩进/规范/注释 |
| **R (相关性)** | 语义匹配度/指令偏离度/上下文连贯性 |
| **E (效率)** | `E = 1 - T_now/T_max`（T_now: 当前耗时，T_max: 最大容忍耗时） |
| **C (资源开销)** | `C = Mem_used/Mem_total + CPU_used/100` |

## 六、极简公式（嵌入式/轻量客户端）

### 单模型版
```
S = λ · (0.6A + 0.3F + 0.1(1-C))
```

### 级联模型版
```
S = η · λ · (0.6A + 0.3F + 0.1(1-C))
```

## 七、使用命令

```bash
# 基础使用
hermes skills call apex-llm-unified-formula score \
  --scenario code_tasks \
  --metrics A=0.95 F=0.90 R=0.88 E=0.92 C=0.35

# 列出可用场景
hermes skills call apex-llm-unified-formula list-scenarios

# 查看当前配置
hermes skills call apex-llm-unified-formula show-config
```

## 八、集成点

| 集成点 | 说明 |
|--------|------|
| hermes self-evaluation | 模型自我评估打分 |
| 模型自动择优 | 选择最高 S<sub>total</sub> 的模型 |
| Prompt 迭代 | 根据得分优化 Prompt |
| 参数调优 | temperature/top_p/context_length 优化 |
| 本地LLM调度 | 根据 C 资源开销动态分配任务 |
